Planned Tools

For this project, Python and R will likely be used to preprocess the data into a format suitable for use in D3. A prototype may be created in Tableau, in conjunction with Excel and/or other tools. Other JS libraries, such as dimple and TopoJSON may also be investigated.

Planned Techniques

My hope for the project is to implement the following techniques:

  1. Projections/Geography: Given that the dataset scope encompasses 197 countries, I hope to make use of the global topographic projection files that are available online. This would provide exposure to using geo-location maps within the D3 framework--something that I feel will be of use after the program. Providing a way for users to interact with a map projection to access the data would be great; however, I plan to at least use these maps navigationally for the project. Whether this manifests as a choropleth or as a navigation section will depend on the amount of work needed to get the desired effect.
  2. Small Multiples: The data also has annual consumption by alcohol type. Given the number of data points and categories, I think small multiples would lend itself well here. Having a single view of all the countries' consumption rates will make understanding all the data much easier. My learning goal for this technique would be to balance the data:ink ratio while preventing user data overload.
  3. Animated Bubble Chart: Similar Bostock's infamous reproduction of Hans Rosling's Wealth & Health of Nations chart, I think this data lends itself well to this kind of representation over time. Whether through this technique or something similar, my plan is to animate the changes of consumption--by type and by country--over the provided time period. The WHO website has more historical data and, if the visualization makes sense, I may get more data specifically to expand on the time component. Here, the learning goal is to understand how to create an automatic time series animation that allows for interactivity.

Planned Interaction

Of the many possible interactions, I intend to implement the following in a linked, unified fashion:

  • Filtering: by country, alcohol category, year, and/or continent
  • Zooming>: in order to zoom into a region of the globe/projection
  • Brushing: if there are any time series components, it will include this for detail views
  • Sorting: here, again, users should be able to sort by country, region, year, or category
